# 复核策略

置信状态：

- `confirmed`：可作为内部问答事实使用。
- `inferred`：模型或Agent推断，需要复核。
- `uncertain`：低置信度，只进入报告或复核区。
- `pending`：等待人工确认。

内部问答：

- 只回答 `confirmed` 内容。
- 待复核内容不作为事实回答。
- 如果问题完全依赖待复核内容，回答“无法给出已确认答案”。

内容生成：

- 可以用待复核内容生成草稿。
- 草稿必须保留复核标记。

回写：

- 只写回 `raw/00_知识库核心资料/`。
- Codex 只能生成补丁，必须经人工确认。
- 不直接修改 `graphify-out/graph.json` 或 Agent知识包。
